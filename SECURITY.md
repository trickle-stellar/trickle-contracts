# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within Trickle, please send an email to **[YOUR EMAIL HERE]**. All security vulnerabilities will be promptly addressed.

**Please do not report security vulnerabilities through public GitHub issues.**

### What to include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix or mitigation**: Depends on severity, typically within 2 weeks

## Scope

This security policy applies to:

- `trickle-contracts` — Soroban smart contracts
- `trickle-backend` — NestJS API server
- `trickle-frontend` — Next.js web application

## Bug Bounty

We are currently in pre-launch. A formal bug bounty program will be announced at launch.

## Preferred Languages

We prefer vulnerability reports in English.
